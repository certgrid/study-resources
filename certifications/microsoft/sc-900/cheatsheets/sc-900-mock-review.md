# SC-900 Mock Review

29 original scenario drills in shuffled order, weighted like the official exam: domain 3 (security solutions) is the heaviest, then domain 2 (Entra), domain 4 (compliance), and domain 1 (concepts). These are original drills written for this pack; they are not real exam questions. Answer each one before looking at the answer guide, then log wrong answers using the Score Booster Routine in the README.

## Drills

1. A security team must collect sign-in logs from Microsoft 365, firewall logs from a Palo Alto appliance, and audit logs from AWS into one place, correlate them, and automatically open a ticket when a detection fires. Which Microsoft service fits?
2. Contoso wants users to sign in without extra prompts from the office network, but require MFA whenever they sign in from anywhere else. Which capability implements this?
3. A banking regulator asks Contoso for Microsoft's independent audit reports and ISO certification documents covering the cloud services Contoso uses. Where does Contoso get them?
4. Contoso moves its HR system to a SaaS application. The vendor says "we handle security now" and the HR team wants to disband its access management process. What responsibility can never transfer to the SaaS provider?
5. An admin needs to open an RDP session to an Azure VM for maintenance, but company policy forbids assigning public IP addresses to VMs. What should they use?
6. A database engineer needs the Global Reader role for about two hours each month to run a report, and the security team wants an approval step plus a full activation history. Which capability fits?
7. A public-facing web app on Azure keeps receiving requests with SQL injection and cross-site scripting payloads. Which service is designed to filter these attacks?
8. Contoso must stop emails containing customer credit card numbers from being sent to external recipients, without blocking normal mail. Which Microsoft Purview capability does this?
9. Overnight, Microsoft's threat intelligence flags that a user's credentials appeared in a published breach dump. Which service raises this as a risk, and what kind of risk is it?
10. The IT director suspects employees are using unsanctioned file-sharing SaaS apps. The team needs to discover which cloud apps are in use and control risky sessions. Which Defender service applies?
11. A developer asks whether to encrypt user passwords in the database so they can be decrypted for support cases. What should the passwords be protected with instead, and why?
12. Sensors on Contoso's domain controllers detect a pass-the-hash attempt and unusual lateral movement inside the on-premises Active Directory environment. Which service produced these detections?
13. Legal documents must be marked "Highly Confidential", encrypted, and watermarked, and the protection must follow the file when it is emailed outside the company. Which Purview capability applies?
14. Every quarter, department managers must confirm that their team members still need membership in a finance application group, with stale access removed automatically. Which capability fits?
15. Contoso wants to protect Exchange Online mailboxes from phishing links and malicious attachments, and to run simulated phishing campaigns against employees. Which Defender service fits?
16. The compliance team needs to assess Contoso's readiness for GDPR, see a risk-weighted score, and get a list of suggested improvement actions with assigned owners. Which tool fits?
17. An Azure web app must read secrets from Azure Key Vault, and the security team requires that no credentials are stored in code or configuration. What should the app use to authenticate?
18. A cloud engineer wants a simple rule on a subnet: allow inbound TCP 443 and deny everything else, filtered by source IP, port, and protocol. Which Azure feature is the lightest fit?
19. Finance invoices must be kept for exactly seven years and then deleted automatically. Editing the invoices during that period is still allowed. Which Purview capability fits?
20. A laptop shows ransomware-like behavior. The SOC wants to isolate the device from the network remotely and run an automated investigation on it. Which Defender service does this?
21. A consultant from Fabrikam needs to collaborate in a Contoso Teams channel using her existing Fabrikam work account, with no new credentials issued by Contoso. Which capability enables this?
22. An application team needs one managed place to store TLS certificates, API keys, and database connection strings, with access controlled and logged. Which Azure service fits?
23. HR reports that an employee who resigned last week has been downloading unusually large volumes of confidential files. The team wants a solution that detects this pattern while keeping the user pseudonymized for reviewers. Which Purview solution fits?
24. Contoso and its subsidiary run separate identity providers. Users in the subsidiary should access Contoso apps by signing in with their own provider, based on a trust between the two. Which identity concept is this?
25. The CISO wants a continuously updated assessment of misconfigurations across Azure and AWS resources, expressed as a secure score with prioritized recommendations. Which capability fits?
26. A vendor proposes "MFA" consisting of a password plus three security questions. Does this qualify as multifactor authentication?
27. A threat hunter wants to research a suspected attacker group: its known infrastructure, indicators of compromise, and threat actor profile. Which Defender service is built for this?
28. Compliance needs to know exactly who accessed a specific SharePoint file six months ago and what they did with it. Which Purview solution answers this?
29. Contoso deploys an AI assistant that reads company data on its own schedule. Security wants the assistant to have its own governed, auditable identity instead of borrowing a user account. Which Entra capability fits?

## Answer guide

1. Microsoft Sentinel (domain 3). SIEM collects and correlates logs from Microsoft AND third-party sources, and SOAR playbooks automate responses like ticket creation. Defender XDR is tempting but it only correlates Microsoft 365 signals and does not ingest arbitrary third-party logs.
2. Conditional Access (domain 2). Location is a signal; the policy grants access from trusted locations and requires MFA elsewhere. Security defaults are tempting but they apply MFA behavior tenant-wide and cannot vary by location.
3. The Service Trust Portal (domain 4). It publishes Microsoft's own audit reports, certifications, and compliance documentation. Compliance Manager is tempting but it assesses Contoso's posture, not Microsoft's evidence for regulators.
4. Data, identities, and accountability (domain 1). In the shared responsibility model the customer always retains responsibility for its data, identities, and devices, even in SaaS. "The vendor handles everything" is the trap; only platform layers transfer.
5. Azure Bastion (domain 3). It provides RDP/SSH through the Azure portal over TLS with no public IP on the VM. An NSG rule is tempting but it still requires some reachable endpoint and does not remove the public exposure.
6. Microsoft Entra Privileged Identity Management (domain 2). PIM gives just-in-time, time-bound activation with optional approval and a full audit trail; it requires Entra ID P2. Access reviews are tempting but they recertify existing access periodically rather than granting temporary elevation.
7. Azure Web Application Firewall (domain 3). WAF is purpose-built for web attacks like SQL injection and XSS. Azure Firewall is tempting but it is a network-level firewall, not an HTTP-layer attack filter.
8. Data loss prevention (domain 4). DLP policies detect sensitive information types such as credit card numbers and block or warn on outbound sharing. Sensitivity labels are tempting but they classify and protect content; they do not by themselves stop an email from leaving.
9. Microsoft Entra ID Protection, as user risk (domain 2). Leaked credentials indicate the ACCOUNT is likely compromised, which is user risk; sign-in risk is the tempting wrong pick but it scores a single suspicious authentication attempt instead.
10. Microsoft Defender for Cloud Apps (domain 3). As a cloud access security broker it discovers shadow IT and applies session controls to SaaS use. Defender for Endpoint is tempting because the traffic comes from devices, but its job is device protection, not SaaS visibility.
11. Hashing (domain 1). Passwords should be stored as one-way hashes that cannot be reversed, so there is nothing to decrypt even for support. Encryption is the trap: it is reversible by design, which is exactly what you do not want for stored passwords.
12. Microsoft Defender for Identity (domain 3). Its sensors on domain controllers detect on-premises AD attacks like pass-the-hash and lateral movement. Entra ID Protection is the classic tempting answer, but it scores cloud sign-in and user risk, not on-premises AD attacks.
13. Sensitivity labels (domain 4). A sensitivity label classifies the document and applies protection such as encryption and watermarks that travel with the file. Retention labels are tempting but they govern how long content is kept, not how it is protected.
14. Access reviews (domain 2). "Periodically confirm who still needs access" with automatic removal of stale access is exactly what access reviews do. PIM is tempting but it handles just-in-time privileged elevation, not recurring recertification.
15. Microsoft Defender for Office 365 (domain 3). Safe Links, Safe Attachments, and attack simulation training protect email and collaboration. Defender for Cloud Apps is tempting because Exchange Online is SaaS, but email threat protection belongs to Defender for Office 365.
16. Compliance Manager (domain 4). It provides assessments against regulations like GDPR, a risk-weighted compliance score, and improvement actions. Microsoft Secure Score is the trap: it measures security posture, not regulatory compliance posture.
17. A managed identity (domain 2). Azure manages the credential lifecycle automatically, so no secret ever appears in code or configuration. A service principal with a client secret is tempting but it reintroduces exactly the stored credential the requirement forbids.
18. A network security group (domain 3). NSGs filter traffic by source/destination IP, port, and protocol at the subnet or NIC level. Azure Firewall is tempting but it is a full managed firewall service; it is heavier than needed for basic five-tuple rules.
19. Retention labels or a retention policy (domain 4). Retention keeps content for a defined period and can delete it afterward, while the content stays editable. Records management is the trap: declaring records adds immutability restrictions the scenario does not ask for.
20. Microsoft Defender for Endpoint (domain 3). EDR capabilities include isolating a device from the network and automated investigation and remediation. Defender for Cloud is tempting because of the similar name, but it protects cloud workloads, not corporate laptops.
21. B2B collaboration with external identities (domain 2). The guest signs in with her home organization's credentials under a B2B invitation. Creating a new Contoso user account is the trap; it duplicates credentials and defeats the purpose of external identities.
22. Azure Key Vault (domain 3). It centrally stores and controls secrets, keys, and certificates with access policies and logging. A storage account is tempting for "storing" things, but it lacks the secret lifecycle, access model, and HSM backing Key Vault provides.
23. Insider risk management (domain 4). It correlates signals such as a resignation date with risky activity like mass downloads, and pseudonymizes users by default. eDiscovery is tempting but it collects evidence for a known case; it does not detect risky behavior patterns.
24. Federation (domain 1). Federation is a trust between identity providers so users authenticate with their own provider and access the other's resources. Hybrid identity synchronization is the trap: that copies accounts to the cloud rather than trusting another provider.
25. Cloud Security Posture Management in Microsoft Defender for Cloud (domain 3). CSPM continuously assesses Azure and multicloud resources and produces a secure score with recommendations. Microsoft Secure Score in the Defender portal is tempting but it measures Microsoft 365 posture, not Azure/AWS resource configuration.
26. No (domain 2). A password and security questions are both "something you know", so only one factor category is present. The trap is counting the number of prompts; MFA requires two or more DIFFERENT categories (know, have, are).
27. Microsoft Defender Threat Intelligence (domain 3). Defender TI provides threat actor profiles, indicators of compromise, and infrastructure analysis for research. Threat analytics in the Defender portal is tempting but it reports on threats relevant to your tenant rather than serving open-ended actor research.
28. Audit solutions in Microsoft Purview (domain 4). The audit log records user and admin activities and supports searching who did what and when; Premium extends retention for older events. eDiscovery is tempting but it finds and preserves content for legal matters; it does not reconstruct activity history.
29. Agent ID (domain 2). Agent ID gives an AI agent its own identity in Microsoft Entra so its access can be scoped, governed, and audited. A workload identity for a generic app is close but the trap is sharing a user account, which destroys accountability; the outline calls out agent ID for AI agents specifically.
