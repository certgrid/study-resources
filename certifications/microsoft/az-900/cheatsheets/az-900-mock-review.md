# AZ-900 Mock Review

This is the final cumulative pass. The drills below span all three domains in shuffled order, exactly like the real exam. Each drill is an original scenario written for this pack; none are real exam questions.

How to use this file:

1. Answer all drills on paper before looking at the answer guide.
2. Score yourself honestly. Aim for 25 or more correct out of 29.
3. For every miss, label the reason (vocabulary, service confusion, governance confusion, cost/monitoring confusion) and re-read the matching section listed in the [Coverage Matrix](../coverage-matrix.md).

## Drills

1. A finance controller needs to see how much each department spent on Azure last month across 40 subscriptions. The resources are already deployed and running. Which combination makes the report possible?
2. A legal team must retain signed contracts for seven years. Retrieval requests are rare, and waiting several hours for a document is acceptable. Which Blob Storage access tier fits best?
3. A startup CFO notices that after moving to Azure, the company stopped making large upfront hardware purchases and now pays a monthly bill that varies with usage. Which financial shift does this describe?
4. A compliance officer must guarantee that no one in the company, in any current or future subscription, can deploy resources outside EU regions. Existing role assignments must not change. What should be configured, and at which scope?
5. A trading application must keep running even if an entire Azure datacenter fails, but latency requirements mean all instances must stay in one region. What should the deployment use?
6. On an Azure virtual machine running Windows Server, who applies OS security patches? And on Azure SQL Database, who patches the underlying platform?
7. After a junior administrator accidentally deleted a production storage account, the team wants deletion of critical resources blocked even for users with the Owner role, without removing anyone's permissions. What should they apply?
8. A partner company's employees need to sign in to your procurement app using their own organization's credentials, without you creating and managing passwords for them. Which capability fits?
9. A hospital keeps patient records in its own datacenter to satisfy regulators but hosts its public appointment-booking website in Azure. Which cloud deployment model is this?
10. A media company must move a 900 TB video archive into Azure within two weeks, but the office uplink is only 100 Mbps. Which transfer option fits?
11. The operations team wants to write one query that searches 30 days of collected logs from 200 VMs in a single workspace. Which Azure Monitor capability do they need?
12. A developer needs an image-resize job that runs only when a photo is uploaded to storage, and the company refuses to pay for idle compute between uploads. Which service fits?
13. A retailer's website automatically adds instances during a flash sale and removes them afterward, with no administrator involved. Which cloud characteristic is this?
14. A migration lead must present the three-year cost of keeping workloads in the company datacenter versus moving them to Azure, including power, space, and staffing. Which tool builds this comparison?
15. Security policy says sign-ins from outside the corporate network must complete an extra verification challenge, while sign-ins from compliant devices in the office proceed normally. Which Microsoft Entra capability enforces this?
16. A company runs servers on-premises, in another public cloud, and in Azure. The platform team wants to project all of them into Azure so they can apply Azure Policy and see them in one management plane. Which service enables this?
17. A department adopts a SaaS email and collaboration suite. The IT manager claims the company now has no responsibilities at all. What does the customer still manage?
18. Two virtual networks in different Azure regions must exchange traffic privately without sending it over the public internet or deploying gateways. Which option fits?
19. A platform team wants dev and test environments deployed from files kept in version control, so every deployment is identical and reviewable. Which approach and Azure capability fit?
20. An ISV must install a custom kernel driver and control the exact OS build for its application. Which compute option is required, and which service model is it?
21. A team wants advance email notice when Microsoft schedules maintenance that will affect their specific VMs, not a public status page for all of Azure. Which tool provides this?
22. An auditor requires that storage data survive the loss of an entire Azure region. Which redundancy options satisfy this?
23. During a rolling failure of one web server instance, users never notice because traffic shifts to the remaining healthy instances. Which cloud benefit is being demonstrated?
24. A legacy application running on several VMs needs a shared folder that all of them mount over SMB. Which storage service fits?
25. Before anything is deployed, the CFO wants a monthly cost estimate for a planned AKS cluster, a database, and outbound bandwidth. Which tool produces it?
26. A new CISO wants a secure score, hardening recommendations, and threat protection across all subscriptions. Which service provides this?
27. A user signs in with a FIDO2 security key, then tries to delete a resource group and is denied because they hold only the Reader role. Which two concepts occurred, and in what order?
28. A data governance officer must discover, classify, and map sensitive data held in Azure SQL databases and on-premises file shares for a compliance program. Which service fits?
29. A public web application must route requests to different backend pools based on the URL path and inspect HTTP traffic at Layer 7. Which service fits?

## Answer guide

| #   | Answer                                                               | Why it is right                                                                                                                             | Why the tempting choice is wrong                                                                                                                      |
| --- | -------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Tags plus Microsoft Cost Management                                  | Tags label resources by department, and Cost Management analyzes actual spend grouped by those tags.                                        | Pricing Calculator is wrong because it estimates future cost; it cannot report on money already spent.                                                |
| 2   | Archive tier                                                         | Archive has the lowest storage cost and is built for long-term data where delayed, offline retrieval is acceptable.                         | Cool or Cold tiers are wrong because they cost more; the scenario explicitly tolerates hours of retrieval delay.                                      |
| 3   | Shift from CapEx to OpEx (consumption-based spending)                | Large upfront purchases are capital expenditure; a variable monthly bill for what you use is operational expenditure.                       | Elasticity is wrong because it describes automatic capacity change, not the financial model of the spending.                                          |
| 4   | Azure Policy assigned at the management group scope                  | Policy controls what can be deployed (allowed locations), and management group scope inherits down to all current and future subscriptions. | RBAC is wrong because it controls who can act, not where resources may be created, and the scenario forbids permission changes.                       |
| 5   | Availability zones                                                   | Zones are physically separate datacenters inside one region, so a full datacenter failure is survived without leaving the region.           | A region pair is wrong because it spans two regions, which breaks the single-region latency requirement.                                              |
| 6   | Customer patches the VM OS; Microsoft patches Azure SQL Database     | A VM is IaaS, so the customer owns the OS. Azure SQL Database is PaaS, so Microsoft manages the platform and OS underneath.                 | "Microsoft patches both" is wrong because IaaS leaves OS management, including patching, with the customer.                                           |
| 7   | A Delete resource lock                                               | Locks block deletion (or changes) regardless of the user's RBAC role, so even Owners must remove the lock first.                            | Tightening RBAC is wrong because it changes permissions, which the scenario forbids, and Owners could still delete.                                   |
| 8   | Microsoft Entra External ID business-to-business (B2B) collaboration | External identities let partner users sign in with their own credentials as guests, with no passwords managed by you.                       | B2C is wrong because it serves consumer-facing apps and customer identities, not partner organization staff.                                          |
| 9   | Hybrid cloud                                                         | Hybrid combines on-premises (private) infrastructure with public cloud services in one solution.                                            | Private cloud is wrong because part of the workload (the website) runs on shared public cloud infrastructure.                                         |
| 10  | Azure Data Box                                                       | Data Box is a physical appliance shipped to you, built exactly for huge datasets where network transfer would take too long.                | AzCopy is wrong because 900 TB over a 100 Mbps link would take years, far beyond the two-week deadline.                                               |
| 11  | Log Analytics (an Azure Monitor capability)                          | Log Analytics is the workspace and query language for searching collected log data across many resources.                                   | Azure Advisor is wrong because it produces best-practice recommendations; it does not store or query logs.                                            |
| 12  | Azure Functions                                                      | Functions is serverless, event-driven compute: code runs when the upload event fires and you pay only for execution time.                   | An always-on VM or App Service plan is wrong because it bills for idle time between uploads.                                                          |
| 13  | Elasticity                                                           | Elasticity is the automatic addition and removal of capacity in response to real-time demand.                                               | Scalability alone is wrong because it only says capacity can change; the keyword "automatically, no administrator" means elasticity.                  |
| 14  | Total Cost of Ownership (TCO) Calculator                             | The TCO Calculator compares full on-premises costs, including power, space, and labor, against the equivalent Azure cost.                   | Pricing Calculator is wrong because it only estimates planned Azure services; it knows nothing about datacenter costs.                                |
| 15  | Microsoft Entra Conditional Access                                   | Conditional Access allows, blocks, or challenges sign-ins based on signals such as location, device state, and risk.                        | Simply enabling MFA for everyone is wrong because the requirement is conditional: only risky sign-ins get the extra challenge.                        |
| 16  | Azure Arc                                                            | Arc projects on-premises and other-cloud servers into Azure Resource Manager so Azure Policy and one management plane apply to them.        | Azure Migrate is wrong because it moves workloads into Azure; the scenario wants to manage them where they already run.                               |
| 17  | Users, data, and access settings                                     | Under shared responsibility, even SaaS customers manage their accounts, information, and who can access it.                                 | "Nothing" is wrong because responsibility is shared at every service level; only infrastructure and the app shift to Microsoft.                       |
| 18  | Virtual network peering (global VNet peering)                        | Peering connects two VNets privately over the Microsoft backbone, works across regions, and needs no gateway.                               | VPN Gateway is wrong because it adds gateway cost and encrypted tunneling that is unnecessary for private VNet-to-VNet traffic.                       |
| 19  | Infrastructure as code using ARM templates (or Bicep)                | Declarative template files in version control give repeatable, reviewable, identical deployments through Azure Resource Manager.            | Portal deployment with written run-books is wrong because manual clicks drift and cannot guarantee identical environments.                            |
| 20  | Azure Virtual Machines, which is IaaS                                | Only IaaS gives control of the OS build and kernel, which installing a custom driver requires.                                              | App Service is wrong because PaaS abstracts the OS away, so you cannot touch kernel drivers or the OS build.                                          |
| 21  | Azure Service Health                                                 | Service Health gives personalized notice of outages and planned maintenance affecting your specific resources and lets you set alerts.      | The public Azure Status page is wrong because it is global and generic; it cannot tell you which of your VMs are affected.                            |
| 22  | GRS or GZRS (geo-redundant options)                                  | Geo-redundant storage copies data to the paired region, so a full regional outage does not destroy it.                                      | ZRS is wrong because zone redundancy protects against datacenter failure inside one region, not the loss of the whole region.                         |
| 23  | High availability                                                    | High availability keeps the service running during component failure, here by shifting traffic to healthy instances.                        | Disaster recovery is wrong because DR is about restoring service after a major disruption, not riding through a single failure.                       |
| 24  | Azure Files                                                          | Azure Files provides managed file shares mounted over SMB (or NFS), exactly what multiple VMs sharing a folder need.                        | Blob Storage is wrong because it is object storage accessed over HTTP APIs; it is not a mountable SMB file system.                                    |
| 25  | Pricing Calculator                                                   | The Pricing Calculator estimates monthly cost for planned Azure services before anything is deployed.                                       | Cost Management is wrong because it analyzes actual spend of deployed resources; nothing exists yet to analyze.                                       |
| 26  | Microsoft Defender for Cloud                                         | Defender for Cloud provides secure score, security recommendations, and workload threat protection across subscriptions.                    | Azure Advisor is wrong because its security suggestions are surfaced from Defender for Cloud; it has no secure score or threat protection of its own. |
| 27  | Authentication first, then authorization                             | The FIDO2 sign-in proved identity (authentication); the RBAC Reader check then decided the delete was not allowed (authorization).          | The reverse order is wrong because Azure cannot evaluate what you may do until it first knows who you are.                                            |
| 28  | Microsoft Purview                                                    | Purview is the data governance service for discovering, classifying, and mapping sensitive data across cloud and on-premises estates.       | Defender for Cloud is wrong because it hardens and protects workloads; it does not catalog and classify data estates.                                 |
| 29  | Application Gateway                                                  | Application Gateway is the Layer 7 load balancer that routes by URL path and understands HTTP/HTTPS traffic.                                | Azure Load Balancer is wrong because it works at Layer 4 (TCP/UDP) and cannot see URL paths inside HTTP requests.                                     |
